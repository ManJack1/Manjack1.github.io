## Getting Started {#getting-started}


### Interactive Sessions {#interactive-sessions}

什么是交互式会话？就是在python的命令行界面输入代码，python会立即执行并返回结果。这种方式非常适合测试一些简单的代码片段或者进行一些快速的计算。
比如我们举个例子

```python
>>> 1 + 2
3
```

我们在符号 `>>>` 后面输入代码 `1+2` python会立即执行并返回结果 `3` 。


### First Example {#first-example}

python对于很多需要计算以及编程的任务都提供来大量的包来使用，比如 `mathlib` 、 `turtle` 等等，这些包提供了很多函数来帮助我们完成各种各样的任务。

```python
>>> from urllib.request import urlopen
```

在这个例子里面 是一个 `import`  语句，import语句的作用是导入一个模块或者包，这样我们就可以使用这个模块或者包里面的函数了。

在广义上来说，计算机程序由两部分指令组成：

-   计算某种值的指令
-   执行某个操作

其中计算某种值的指令叫做 **表达式** ，执行某个操作的指令叫做 **语句** 。
本章我们主要介绍的是几种类型的表达式和语句。

**赋值语句**

```python
>>> shakespeare = urlopen('http://composingprograms.com/shakespeare.txt')
```

在这个例子中， `shakespeare` 是一个变量， `urlopen('http://composingprograms.com/shakespeare.txt')` 是一个表达式，整个 `shakespeare = urlopen('http://composingprograms.com/shakespeare.txt')` 就是一个赋值语句。

赋值语句的作用是将一个表达式的值赋给一个变量，变量就像一个容器，可以用来存储数据。我们可以通过变量来访问存储的数据。

```python
>>> words = shakespeare.read().decode('ascii').split()
```

**对象**
对象是编程语言中的一个重要的概念，它将数据和操作数据的逻辑捆绑在一起。在对象中我们含有数据成员和方法成员，数据成员是用来存储数据的变量，方法成员是用来操作数据的函数。
比如

```python
>>> {w for w in words if len(w) == 6 and w[::-1] in words}
{'redder', 'drawer', 'reward', 'diaper', 'repaid'}
```

**解释器**
解释器是一个程序，它可以执行我们写的代码。我们在python的命令行界面输入代码，python解释器会立即执行并返回结果。解释器的作用是将我们写的代码翻译成计算机能够理解的语言，然后执行这些代码。

<div class="note">

最后我们做一个总结：函数是对象，对象是函数，而解释器是两者的桥梁。

</div>


### Error {#error}

面对错误，程序会比人死板很多，即使是很小的拼写错误或者格式的改变也会导致意外的输出和错误。

学会解释错误并且分析出错误的原因称之为 **调试** 。调试的一些基本原则：

1.  逐步测试：每个写得好的程序都是由小的、模块化的组件组成的，这些组件可以单独测试。尽快尝试你写的所有内容.
2.  隔离错误：语句的输出中的错误通常可以归因于特定的模块化组件。在尝试诊断问题时，在尝试纠正它之前，将错误追溯到你能找到的最小的代码片段。
3.  检查你的假设：解释器确实会逐字执行你的指令——不多也不少。当某些代码的行为与程序员认为（或假设）的行为不匹配时，它们的输出就会出乎意料。了解你的假设，然后专注于验证你的假设是否真的成立。
4.  咨询他人：你并不孤单！如果你不理解错误消息，可以问朋友、导师或搜索引擎。如果你已经隔离了错误，但不知道如何纠正它，可以请别人看看。在小组解决问题的过程中，会分享很多宝贵的编程知识。
