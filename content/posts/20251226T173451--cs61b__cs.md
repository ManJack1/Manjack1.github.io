### <span class="section-num">1.2</span> Bit {#bit}

你电脑所有信息都是由一串0和1组成的，这些0和1叫做bit（二进制位）。每一个bit只能是0或者1。8个bit组成一个字节(byte)。

有一个有趣且实用的设计，比如在计算机中 72和 H存储都为01001000。java以及其他语言是通过定义的类型来区分这些bit代表的意义的。

```java
char letter = 'H'; // 代表字符H
int number = 72;   // 代表数字72
```

虽然它们的bit表示一样，但是它们的类型不同，所以计算机知道它们代表不同的意义。


### <span class="section-num">1.3</span> Declaring a Variable {#declaring-a-variable}

-   在 java中不同的类型有不同的比特大小，声明的时候 java会为你找到合适的内存块，并将变量名绑定到内存块上。
-   在 java中并没有像指针那样，可以让你知道内存地址的方法。

-   java 的初始化不会给变量一个随机值，而是给一个默认值。
    -   int 默认值是0
    -   double 默认值是0.0
    -   boolean 默认值是 false
    -   object 默认值是 null

java有八大基本类型：byte, short, int, long, float, double, char, boolean

<div class="NOTE">

在语言中变量名占用内存空间吗？
在多数编程语言中，变量名本身并不会在运行时占用内存空间。编译后的程序通常将变量名替换为内存地址或寄存器。因此，变量名只在 **源代码和调试信息中存在** 。
但是，在解释型语言（如 Python 或 JavaScript）中，变量名可能存储在符号表或字典中，用来动态管理变量。此时，变量名会占用一些内存，但相对于数据本身，通常是可以忽略的。

</div>


### <span class="section-num">1.4</span> Reference Types {#reference-types}

上述我们说了基本的八大类型，其他的所有类型都不是基本类型，而是引用类型(reference type)。 引用类型的变量存储的是对象在内存中的地址，而不是对象本身。

```java
public static class Walrus{
    public int weight;
    public double tuckSize;

    public Walrus(int w, double ts)
    {
        weight = w;
        tuckSize = ts;
    }
}
```

-   在上述的代码中， `new` 关键字会在内存中创建一个 Walrus 对象，并返回该对象的内存地址。这个地址会被赋值给变量 a。
-   在对象被创建的时候，walrus其实是32bit+64bit,但是实际上还会有一些额外的开销，比如对象头(object header)等，所以实际占用的内存会比这个大一些。


### <span class="section-num">1.5</span> Refernce Variable Decalration {#refernce-variable-decalration}

声明一个引用类型的变量，java都会分配一个64位数的内存空间来存储对象的地址。

```java
Walrus a; // 声明一个引用类型变量a
a = new Walrus(1000,8.3); // 创建一个Walrus对象，并将地址赋值给a
```

注意 a是存储walrus对象地址的变量，而不是walrus对象本身(其数据)。


### <span class="section-num">1.6</span> parameter Passing {#parameter-passing}

对于不是基本类型的参数传递，java使用的是引用传递(reference passing)。这意味着当你将一个对象作为参数传递给一个方法时，传递的是对象的引用（地址），而不是对象的实际数据。

```java
public static double average(double a, double b) {
    return (a + b) / 2;
}
```

在上述代码中，average方法接受两个double类型的参数e和b。当调用average方法时，传递的是e和b的值的副本，而不是它们的引用。所以如果在函数内部改变a和b的值并不会影响到原始变量。但是引用类型是会影响到变量的。

例子：

```java
public class PassByValueFigure {
    public static void main(String[] args) {
        Walrus walrus = new Walrus(3500, 10.5);
        int x = 9;

        doStuff(walrus, x);

        System.out.println(walrus);
        System.out.println(x);
    }

    public static void doStuff(Walrus walrus, int x) {
        walrus.weight -= 100;
        x -= 5;
    }
}
```

在上述代码中，doStuff方法接受一个Walrus对象和一个int类型的参数。当调用doStuff方法时，传递的是walrus对象的引用和x的值的副本。在doStuff方法内部，修改W.weight会影响到原始的walrus对象，因为W引用的是同一个对象。但是修改x不会影响到原始的x变量，因为x是传递的值的副本。


### <span class="section-num">1.7</span> Instantiation of Arrays {#instantiation-of-arrays}

在java中，数组是一种引用类型。当你创建一个数组时，实际上是在内存中分配了一块连续的空间来存储数组元素，并返回该空间的地址。

```java
int[] numbers;
Planet[] planets;

numbers = new int[]{1,2,3,4,5};
planets = new Planet[8];
```

在上述代码中，numbers是一个引用类型变量，它可以存储一个int数组的地址。planets是一个引用类型变量，它可以存储一个Planet对象数组的地址。
当你使用 `new` 关键字创建一个数组时，java会在内存中分配一块连续的空间来存储数组元素，并返回该空间的地址。这个地址会被赋值给引用类型变量numbers或planets。


### <span class="section-num">1.8</span> The Law of the Broken Futon {#the-law-of-the-broken-futon}

在java中，当你创建一个对象时，java会在内存中分配一块空间来存储对象的数据，并返回该空间的地址。如果你没有保存这个地址，那么你将无法访问这个对象，这就像是一个破碎的沙发(futon)，你无法坐在上面。


### <span class="section-num">1.9</span> `==` vs Arrays.equals {#vs-arrays-dot-equals}

`==` 从位的角度来看，它比较的是两个引用类型变量所存储的地址是否相同。
比如在代码中：

```java
int[] x = {1, 2, 3};
int[] y = {1, 2, 3};

System.out.println(x == y); // false
system.out.println(Arrays.equals(x, y)); // true
```

虽然它们的内容相同，但是位不同,所以结果是flase。
而 Arrays.equals 方法比较的是两个数组的内容是否相同。


### <span class="section-num">1.10</span> IntLists {#intlists}

```java
public class IntList {
    public int first;
    public IntList rest;

    public IntList(int f, IntList r) {
        first = f;
        rest = r;
    }
}
}
```

这种写法只适合在头部插入，如果要在尾部插入就需要遍历到整个链表的最底部，非常繁琐。

```java
IntList l = new IntList(1,null);

// 那么就需要遍历到 l.rest.rest = new IntList(3,null);
// 过于繁琐

// 推荐的写法是：
l = new IntList(1,l);
l = new IntList(2,l);
l = new IntList(3,l);
//这样的写法简单，且高效。
```


### <span class="section-num">1.11</span> size and iterativeSize {#size-and-iterativesize}

计算链表的大小可以使用递归和迭代两种方法。


#### <span class="section-num">1.11.1</span> size {#size}

递归求得大小

```java
public int size() {
    if (rest == null) {
        return 1;
    }
    return 1 + rest.size();
}
//不能用this == null,因为根本无法访问到this就报错了
```


#### <span class="section-num">1.11.2</span> iterativeSize {#iterativesize}

迭代求得大小

```java
public int iterativeSize() {
    IntList p = this;
    int totalSize = 0;
    while (p != null) {
        totalSize += 1;
        p = p.rest;
    }
    return totalSize;
}
```


### <span class="section-num">1.12</span> get {#get}

get是获取列表中的元素。

```java
public int get(int x) {
    IntList p = this;
    while (x > 0) {
        p = p.rest;
        x--;
    }
    return p.first;
}
```

### <span class="section-num">6.1</span> 列表的性能问题 {#列表的性能问题}

对于DDLists的链表来说， `get`  要比 `getlast` 要慢很多，因为 `getlast` 只需要访问最后一个节点，而 `get`  需要从头节点开始遍历整个链表，直到找到指定索引的节点，这个问题对于内存不连续的链表都极为普遍。
比如我们的链表长度是100000， `getlast`  只需要访问最后一个节点，而 `get(99999)`  需要访问100000个节点，时间复杂度是O(n)，所以 `getlast`  要比 `get`  快很多。
但如果我们使用数组作为简单的列表实现，那么访问列表的速度速度将会快很多，具体请看[[<https://www.quora.com/Why-does-accessing-an-array-element-take-constant-time>][链表]

### <span class="section-num">6.3</span> 动态调整数组的大小 {#动态调整数组的大小}

![](/ox-hugo/20251231-131243_screenshot.png)
因为我们是数组构成的列表，当数组满的时候，我们是不可以给数组扩容的。所以我们只能创建一个更大的数组，然后将原数组的内容复制到新数组中，最后将新数组赋值给原数组变量。
在java中，我们可以使用 `System.arraycopy` 方法来实现数组的复制。

<div class="NOTE">

System.arraycopy(Object src, int srcPos, Object dest, int destPos, int length)

-   src: 原数组
-   srcPos: 从原数组的哪个位置开始复制
-   dest: 目标数组
-   destPos: 从目标数组的哪个位置开始粘贴
-   length: 复制的长度

</div>

<div class="ep">

练习 2.5.4：尝试实现 addLast(int i) 方法以处理调整大小的数组。

</div>

```java
public void resize(int newCapacity) {
    int[] a = new int[newCapacity];
    System.arraycopy(items, 0, a, 0, size);
    items = a;
}

public int addLast(int x) {
    if (size == items.length) {
        resize(size + 1);
    }
    items[size] = x;
    size += 1;
    return x;
}
```


### <span class="section-num">6.4</span> 分析调成数组大小的性能 {#分析调成数组大小的性能}

对了普通的链表 `addlast` 1000000次的时间都是非常短，但是ArrayList调用 `addlast` 的时间倒是久了很多，主要原因就是在数组满的时候，需要创建一个新的更大的数组，并将原数组的内容复制到新数组中，这个过程是非常耗时的。并且数组满了每次要扩容1个单位也是非常低效的，因为每次扩容都需要复制整个数组的内容。假如我们数组的 `capicifty` 是100，到了100之后，我们要扩容到101，然后是102，103，104，每次都要复制整个数组的内容，累计运行这样的时间复杂度是O(n^2)。

{{< figure src="/ox-hugo/20251231-132512_screenshot.png" caption="<span class=\"figure-number\">Figure 2: </span>数组扩容性能问题" >}}


#### <span class="section-num">6.4.1</span> 改进数组扩容的方法：几何级扩容 {#改进数组扩容的方法-几何级扩容}

一种更好的方法是每次扩容的时候，将数组的大小翻倍，这样就可以减少扩容的次数，从而提高效率。比如当数组满了，我们将数组的大小从100扩容到200，然后是400，800，每次扩容都只需要复制一半的数组内容，累计运行时间复杂度是O(n)。
我们可以通过按倍数增加数组的大小来解决这个问题，而不是按加数增加。也就是说，而不是增加等于某个缩放因子 RFACTOR 数量的内存框：对于该图片来说使用率仅为0.04。

```java
//放弃
public void insertBack(int x) {
    if (size == items.length) {
           resize(size + RFACTOR);
    }
    items[size] = x;
    size += 1;
}

//使用几何级别扩容
public void insertBack(int x) {
    if (size == items.length) {
           resize(size * RFACTOR);
    }
    items[size] = x;
    size += 1;
}
```


#### <span class="section-num">6.4.2</span> 缩小数组 {#缩小数组}

D当我们数组有大量空间未使用的时候，我们应该对数组进行缩小，以节省内存空间。我们可以在删除元素的时候，检查数组的使用率，如果使用率低于某个阈值，就将数组缩小一半。

要解决这个问题，我们也可以在数组看起来空的时候减小其大小。具体来说，我们定义一个"使用率"R，它等于列表的大小除以 items 数组的长度。例如，在下图中，使用率是 0.04。

{{< figure src="/ox-hugo/20251231-133358_screenshot.png" caption="<span class=\"figure-number\">Figure 3: </span>数组使用率" >}}

在研究中当 `R` 小于0.25的时候，我们就将数组缩小一半（见 [Sedgewick &amp; Wayne, \*Algorithms (4th ed.)\*：Resizing Arrays](https://algs4.cs.princeton.edu/13stacks/) 的说明）。

```java
public int removeLast() {
    int last = items[size - 1];
    size = size - 1;
    if (size < items.length / 4) {
        resize(items.length / 2);
    }
    return last;
}
```
